# SignMeet — TF.js Model Directory

This folder is served statically by Vite at `/model/`.

## Required files

| File | Description |
|---|---|
| `model.json` | TF.js model topology (generated by `tensorflowjs_converter` or `model.save()`) |
| `weights.bin` | Binary weight shard referenced by `model.json` |

## Generating the model

Train the gesture classifier with the training script in the repo:

```bash
cd train_model
python train.py
```

`train.py` will:
1. Load the landmark dataset from `data/landmarks.csv`
2. Train a small dense network (63 inputs → 128 → 64 → 5 softmax outputs)
3. Export the Keras model directly to TF.js format:
   ```python
   import tensorflowjs as tfjs
   tfjs.converters.save_keras_model(model, "../frontend/public/model")
   ```

The five output classes correspond to `SIGN_LABELS` in `useSignDetection.js`:

```
index 0 → HELLO
index 1 → THANKS
index 2 → BYE
index 3 → YES
index 4 → NO
```

## Mock mode (no model present)

If `model.json` is missing or fails to load, `useSignDetection` automatically
falls back to **mock detection**:

- `mockSignDetection.js` fires a random sign every 3 seconds
- The full UI pipeline (badges, captions, WebSocket broadcast, TTS) runs
  exactly as it would with a real model
- Browser console shows:
  ```
  [SignDetect] Model not found, using mock detection for development
  ```

Mock mode lets you develop and test the complete front-end experience
before any training data is collected.

## Input format

The model expects a `[1, 63]` tensor:

- 21 MediaPipe hand landmarks × 3 coordinates (x, y, z)
- All values normalised relative to the wrist (landmark 0)
- Coordinate range: approximately −1.0 to +1.0

## Adding new signs

1. Add the label to `SIGN_LABELS` in `useSignDetection.js`
2. Collect landmark data for the new gesture
3. Re-train and re-export the model
4. Replace `model.json` and `weights.bin` in this directory
